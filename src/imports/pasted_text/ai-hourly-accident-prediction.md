## 1. AI-Based Hourly Accident Prediction in NYC Using Weather Data

**배지:** RESEARCH | 2024 한국통신학회 추계종합학술발표회

**기본 정보**
- 소속: Qualcomm Institute, UC San Diego
- 기간: 2024.07 – 2024.08
- 역할: 공동 저자 (경기대학교 대표)
- 스택: `Python` `XGBoost` `RandomForest` `Preprocessing` `KNIME`

---

**배경 및 문제 정의**
뉴욕시는 세계 최고 수준의 교통량으로 인해 매년 20만 건 이상의 교통사고가 발생하며, 이로 인한 인명 피해와 경제적 손실이 막대하다. 기존 선행 연구 대부분이 사고 위치나 시간 정보만을 활용한 데 반해, 주요 사고 유발 요인 중 하나인 **날씨 데이터를 결합한 시간별(Hourly) 예측 연구는 부족**한 상황이었다.

---

**데이터**
- NYC Open Data의 **NYC Motor Vehicle Collisions** (2012.07 ~ 2019.12, COVID 영향 구간 제외)
- Visual Crossing의 **Total Weather Data** (동일 기간, 시간 단위 기상 데이터 23개 특성)
- 두 데이터셋을 결합해 **Constructed Feature Dataset(CFD)** 생성 — 12개 독립변수, 1개 종속변수(시간별 교통사고 건수)

---

**방법론**
7단계 정기 시계열 데이터 전처리 파이프라인 구축
1. 시간별 사고 건수 집계 (Crash Date + Hour → Hourly Car Accidents)
2. COVID 영향 기간(2020년~) 데이터 제거
3. 관련성 낮은 변수 제거 (FeelsLike, CloudCover, SolarRadiation 등)
4. 주중/주말 구분 변수(Weekday) 추가
5. Icon 특성 전처리 (밤/낮, 구름 상태 → Clear 통합)
6. CFD 생성 및 Min-Max 정규화 적용 → CFDN 생성
7. 2012~2018년 학습 / 2019년 테스트 분할

**모델:** Random Forest Regression(RFR) vs. XGBoost — 하이퍼파라미터 최적화 수행

---

**결과**
| 모델 | 데이터셋 | R² | RMSE |
|------|----------|-----|------|
| XGBoost | CFD | **0.782** | **6.586** |
| XGBoost | CFDN | 0.781 | 6.604 |
| RFR | CFD | 0.768 | 6.798 |
| RFR | CFDN | 0.767 | 6.805 |

- CFD 기반 XGBoost 모델이 최우수 성능 달성
- RMSE 6.586은 실제 시간별 사고 최댓값(75건)의 약 9% 수준으로, 표준편차(14.11)보다도 낮음

---

**의의**
시간대별 사고 예측 정확도를 통해 교통 안전 정책 수립 및 비반복적 교통 혼잡 완화에 기여 가능

## 2. 중장년 맞춤형 AI 교육 추천 에이전트 "다시, 봄"

**배지:** 🏆 우수상 | 연세대학교 미래 ICT 서울 지역사회 경험학습 공모전

**기본 정보**
- 기간: 2025.11 – 2025.12
- 팀: 경기대학교 (응용통계학, 경영정보, 산업시스템공학 전공)
- 스택: `Python` `LangChain` `RAG` `FAISS` `OpenAI API`

---

**배경 및 문제 정의**
초고령화 시대를 맞아 중장년층이 최대 경제활동 인구 집단으로 부상했지만, AI 활용 역량 부재로 인한 생산성 격차가 실질적 고용 위기로 이어지고 있다. (대한민국 중장년층 고용 불안정성 OECD 1위)

AI 교육 콘텐츠는 다수 존재하지만 분산·파편화되어 있고, 정보처리 속도 저하 및 인지적 제약이 있는 중장년층에게 이 환경은 치명적이다. 직무 경험과 수준에 맞는 **맞춤형 교육 추천 서비스의 부재**가 핵심 문제였다.

---

**시스템 구조 (파이프라인)**

**① 데이터 수집**
고용24, 서울50플러스, STEP 등 공공·민간 교육 플랫폼에서 강의명, 강의 내용, URL 수집

**② 청킹 및 임베딩**
한국어 문맥 이해 특화 모델 **Ko-SRoBERTa** 기반 임베딩

**③ 벡터 DB 구축**
**FAISS** 기반 벡터 DB에 적재

**④ LangChain 기반 RAG 챗봇**
- 단계형 대화로 사용자 정보 수집 (직무 분야 → 담당 업무 → AI 활용 수준)
- LLM이 응답을 요약해 사용자 프로필 생성 → 검색 쿼리 변환 (할루시네이션 최소화)
- LLM 기반 적합성 검증 모듈: 직무 연관성·난이도 기준으로 강의 적합/부적합 판단 후 추천

**⑤ Flask 웹 애플리케이션** 으로 전체 파이프라인 배포 — 강의명 클릭 시 신청 페이지 연결

---

**기대 효과**

| 차원 | 내용 |
|------|------|
| 개인 | 직무 경험 반영 AI 교육 추천으로 재취업 경쟁력 강화 |
| 기업 | AI 역량 갖춘 중장년 인력 효과적 매칭 |
| 사회 | 교육-일자리 연계를 통한 고용 선순환 기반 마련 |

---

**활용 방안**
- 중장년 대상 맞춤형 AI 교육 추천 서비스 (재취업 지원 도구)
- 재취업·평생학습 플랫폼 연계 허브 (교육 수료자 기업 역제안 매칭)
- 기업 내 중장년 재직자를 위한 리스킬(Reskill) 솔루션 (LMS 기반)

## 3. 웹 소설 – 애니메이션 매체 전환(OSMU)의 사용자 주요 흥미 포인트 도출

**배지:** 기업 프로젝트 | STELLA&

**기본 정보**
- 기간: 2025.09 – 2025.12
- 협업: STELLA& × SODA LAB. (경기대학교 경영정보전공)
- 스택: `Python` `Data Analysis` `OSMU` `Text Mining` `Topic Modeling`

---

**배경 및 문제 정의**
웹소설 IP 기반 애니메이션화가 증가하는 추세이나, 장르별 성공 요인 판단이 제작진의 경험·감각에 의존하고 있었다. STELLA&는 웹소설을 숏폼·미드폼·롱폼으로 영상화하는 B2B 기업으로, **시청자가 실제로 어떤 포인트에서 흥미를 느끼는지**에 대한 정량적 근거가 필요했다.

---

**분석 방법**

**분석 1. 판타지 장르 주요 시청 경험 요인 분석**
- 대상: 이세계 전생물 애니메이션 3개 작품 (LAFTEL 리뷰 총 4,200건)
  - 전생했더니 슬라임이었던 건에 대하여 1기 / 무직전생 PART1 / 거미입니다만, 문제라도?
- 파이프라인: 텍스트 전처리 → **BERTopic** 토픽 모델링 → **BERT** 하이브리드 감성 분석 → 네트워크 분석 (근접 중심성 기준)

**분석 2. 회차별 시청자 리뷰 감성 분석**
- 대상: LAFTEL `<나 혼자만 레벨업>` 자막판 리뷰 (총 706건)
- 방법: **KoELECTRA** 기반 딥러닝 감성 분석 → 회차별 감성 점수(0~1) 시계열 그래프화

---

**주요 결과**

**판타지 장르 토픽 6개 도출:**

| 토픽 | 핵심 내용 |
|------|----------|
| 캐릭터 팬덤 및 애정 | 주인공 외형·성격의 감정적 매력이 핵심 몰입 요소 |
| 조연 서사 및 인물관계도 | 주인공 부재 구간에서 감정 분열 발생 |
| 전투 쾌감 및 긴장감 | 액션씬 만족/아쉬움이 리뷰 주요 분포 |
| 제작 품질 및 만족도 저하 | 작붕·기술적 결함에 대한 냉정한 비판 |
| 장르적 호불호 및 스토리라인 | 클리셰 수용층 vs 양판소 비판층 양분 |
| 정서적 몰입 및 서사적 감동 | 감정적 만족도가 가장 높은 구간 |

**나 혼자만 레벨업 회차별 감성 분석:**
- 최고점: 12화(0.78) — 압도적 액션 퀄리티와 원작 초월 연출로 피날레 효과 극대화
- 최저점: 7화(0.43) — 총집편 편성으로 인한 운영 배신감 → 직접적 이탈 언급 다수
- 특이점: 5화(0.48) — 중반부 전개 속도 저하 구간, 자막판 시청자 흥미 하락 감지

---

**비즈니스 인사이트 제공**

| 현상 | 시청자 반응 | 비즈니스 리스크 |
|------|-----------|--------------|
| 구성 문제 | 총집편 편성 불만 | 흐름 단절 |
| 속도 문제 | 전개 템포 아쉬움 | 시청 지속 시간 감소 |
| 각색 문제 | 원작과의 괴리 | 원작 팬덤 반발 |

## 4. Plannie: GPT 기반 맞춤형 공부 일정 관리 앱

**배지:** 공모전 발표 | 2024 경기 SW 페스타

**기본 정보**
- 기간: 2024.04 – 2024.11
- 팀: 박승연, 박진서, 문예진, 장서진 (경기대학교 소프트웨어중심대학사업단)
- 스택: `Python` `OpenAI API` `Few-shot` `Prompt Eng.`

---

**배경 및 문제 정의**
기존 일정 관리 앱은 단순 알림·캘린더 기능에 그쳐, 학습자의 개별 학습 패턴이나 목표를 반영한 맞춤형 지원이 부재했다. 학생들은 학습 목표·달성 현황을 스스로 추적하는 데 많은 시간을 소비하고 있었다.

---

**핵심 기능**

| 기능 | 내용 |
|------|------|
| GPT Chatbot | 시험 일정·범위 입력 시 맞춤형 학습 계획 자동 생성 |
| 개인화 목표 설정 | Few-shot Prompt로 개별 학습 패턴 반영 |
| 캘린더 + To-Do List | 월별 일정 확인 및 학습 수행 진도 추적 |
| 알림 & 리마인더 | 학습 목표 달성 지원 |

---

**시스템 구조**
Client(프론트엔드) → GitHub → AWS EC2 → Node.js → MongoDB / Amazon RDS
GPT 모델 파인튜닝으로 학생 학습 대화 데이터 기반 효과적 학습 관리 지원

---

**기존 앱과의 차별점**

| 기존 일정 관리 앱 | Plannie |
|-----------------|---------|
| 단순 알림 및 캘린더 | GPT 기반 맞춤형 학습 계획 제안 |
| 개인화 학습 지원 부재 | 학습 패턴 반영한 목표 설정 |
| 수동 진도 확인 | To-Do List + 달성 진도 자동 추적 |

---

**기대 효과**
- 학습 효율성 향상: 개인 요구에 맞는 일정·방법 제공으로 시간 관리 및 성취도 향상
- 편리한 학습 환경: 직관적 UI/UX로 학습 계획 손쉽게 관리
- 향후 확장: 온라인 강의 플랫폼 연계, 공부 타이머 앱 연동 등
